# 听股通 · 产品需求文档（PRD）

> 版本：v2.0 · 日期：2026-04-07

---

## 1. 产品概述

### 产品名称
**听股通**

### 一句话定位
用户通过"听"的方式，实时掌控 A 股行情，不受时间、空间、手机屏幕限制。

### 核心价值
解决上班族、司机、手部不便者等"不方便看屏幕"场景下的 A 股行情获取问题，让用户在开车、开会、运动、做饭等任意场景下，用耳朵代替眼睛。

### 目标用户
- 上班族（没时间盯盘，但关心持仓）
- 司机（驾驶时无法看手机）
- 手部不便人群 / 有视力障碍的投资者
- 超短线交易者（需要实时感知盘面异动）
- 任何想"听"行情而非"看"行情的 A 股投资者

## 2. 播报体系

### 2.1 全量播报类型（共 11 种）

| 编号 | 播报类型 | 触发条件 | 播报格式示例 | 优先级 |
|------|---------|---------|------------|--------|
| A1 | 自选股行情（定时） | 到达播报间隔（30秒/1分钟/5分钟） | `茅台，报1688元，涨2.3%` | P4 例行 |
| A2 | 快速拉升预警 | 5分钟涨幅达到阈值（默认3%，可调） | `⚠️ 拉升！某某股份，5分钟涨了5.2%` | P2 紧急 |
| A3 | 快速下跌预警 | 5分钟跌幅达到阈值（对称拉升阈值） | `⚠️ 下跌！某某股份，5分钟跌了4.1%` | P2 紧急 |
| A4 | 涨停预警 | 个股首次触板（主板±10%/科创/创业±20%） | `🔴 涨停！某某股份，报XX元，涨停！` | P1 最高 |
| A5 | 跌停预警 | 个股首次触跌停价 | `⚠️ 跌停！某某股份，报XX元，已跌停！` | P1 最高 |
| A6 | 炸板预警 | 已涨停个股打开涨停板 | `⚠️ 炸板！某某股份，打开涨停，当前报XX元` | P1 最高 |
| A7 | 板块异动 | 板块涨幅>阈值（默认2%，每天每板块1次） | `📡 板块异动：券商板块涨幅2.3%` | P3 中等 |
| A8 | 大盘异动 | 沪深300/上证/深证涨跌幅超阈值（默认1%） | `注意！大盘大幅波动，上证指数目前涨1.2%` | P2.5 重要 |
| A9 | 成交量异常 | 个股5分钟成交量超昨日均量倍数（默认3倍） | `某某股份成交量异常放大，是昨日均量3倍` | P4 例行 |
| A10 | 集合竞价异动 | 竞价阶段（9:15-9:25）涨跌幅超阈值（默认5%） | `竞价预警：某某股份高开4.2%` | P3.5 重要 |
| A11 | 触及止盈/止损 | 持仓股触及用户预设止盈/止损价格 | `⚠️ 茅台触及您设定的止盈价1700元` | P1 最高 |

### 2.2 用户开关机制

**每个播报类型独立开关（默认全部开启）。**

用户可自由组合开启/关闭任意播报类型，灵活满足个性化需求。

例：用户只关心"涨停"和"自选股行情"，关闭其余所有播报 → 最小打扰模式。

### 2.3 统一优先级体系

```
P0 · 最高（同时触发时按 A4→A5→A6→A11 顺序）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
涨停 / 跌停 / 炸板 / 触及止盈止损
    → 可打断任何播报，同时触发按编号顺序播放

P1 · 紧急
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
快速拉升 / 快速下跌 / 大盘异动
    → 可打断 P2/P3/P4；拉升和下跌不互相打断

P2 · 中等
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
板块异动 / 集合竞价异动
    → 正常排队，不打断同级

P3 · 例行
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
自选股行情 / 成交量异常
    → 基础播报，最低优先级
```

### 2.4 统一调度规则（4条核心）

1. **高级打断低级**：P0 永远可打断任何正在播放的内容
2. **低级不打断高级**：P3 永远不能打断任何播报
3. **同级串行**：同优先级的播报按触发先后进入队列，依次播放
4. **去重防刷**：每个播报类型独立去重，避免重复骚扰

### 2.5 各播报类型去重规则

| 播报类型 | 去重规则 |
|---------|---------|
| 涨停 / 跌停 | 每只股票每天最多 1 次（次日重置） |
| 炸板 | 每只股票每天最多 1 次（封板后再炸重新计） |
| 快速拉升 / 下跌 | 同一只股票 5 分钟窗口内最多 1 次 |
| 板块异动 | 每个板块每天最多 1 次 |
| 大盘异动 | 每类指数（沪深300/上证/深证）每天各最多 3 次 |
| 成交量异常 | 同一只股票 30 分钟内最多 1 次 |
| 集合竞价异动 | 竞价结束后（9:25）自动失效 |
| 触及止盈/止损 | 触发后需用户手动重置或次日自动重置 |
| 自选股行情 | 按用户设定的间隔循环，不去重（正常播报） |

## 3. 功能规格

### 3.1 自选股实时行情播报

**功能描述**
用户添加关注的股票，APP 在后台持续监控并在设定间隔播报实时行情。

**用户操作流程**
1. 进入"自选股"页面
2. 点击"添加股票"，搜索：名称 / 代码 / 拼音首字母
3. 选择股票，确认添加；设置播报间隔（30秒 / 1分钟 / 5分钟）
4. 股票出现在列表；返回首页后自动开始播报

**播报格式**
> `茅台，报1688元，涨2.3%`
> `平安银行，报12.45元，跌了0.8%`

**后台运行**：锁屏、切换APP、手机放口袋均持续播报

---

### 3.2 快速拉升/下跌预警

**配置参数**
- 涨幅/跌幅阈值：默认 3%，最低 3%，可上调（5%、8%、10%）
- 时间窗口：5 分钟（固定）
- 监控范围：全市场所有 A 股

**播报格式**
> `⚠️ 拉升！某某股份，5分钟涨了5.2%，当前报38.6元`
> `⚠️ 下跌！某某股份，5分钟跌了4.1%，当前报28.3元`

---

### 3.3 涨停 / 跌停 / 炸板预警

**涨停触发条件**
- 主板（沪深）：股价达到「昨收盘 × 1.10」
- 科创板 / 创业板：股价达到「昨收盘 × 1.20」
- 每只股票每天首次触板时触发一次

**跌停触发条件**（对称设计）
- 主板：股价达到「昨收盘 × 0.90」
- 科创 / 创业板：股价达到「昨收盘 × 0.80」

**炸板触发条件**
- 曾触发涨停且尚未解除 → 当前价 < 涨停价 × 0.998
- 每只股票每天炸板首次触发时播报

**播报格式**
> `🔴 涨停！某某股份，报110.2元，涨停！`
> `⚠️ 跌停！某某股份，报88.5元，已跌停！`
> `⚠️ 炸板！某某股份，打开涨停，当前报108元`

---

### 3.4 大盘异动监控

**配置参数**
- 阈值：默认 ±1%，可调（0.5%、1%、2%）
- 监控指数：上证指数、深证成指、创业板指（可选）

**播报格式**
> `注意！上证指数大幅上涨，目前涨1.3%`
> `注意！沪深300跌幅扩大，当前跌1.1%`

---

### 3.5 板块异动监控

**配置参数**
- 阈值：默认 2%（固定，不可调）
- 同板块播报频率：每天最多 1 次

**覆盖板块**：约 50 个主要行业板块（券商、银行、半导体、新能源、医药等）

**播报格式**
> `📡 板块异动：券商板块涨幅2.3%`

---

### 3.6 成交量异常监控

**配置参数**
- 倍数阈值：默认 3 倍（相对昨日均量）
- 时间窗口：5 分钟成交量
- 冷却时间：同一股票 30 分钟内不重复

**播报格式**
> `某某股份成交量异常放大，是昨日均量的3倍`

---

### 3.7 集合竞价异动

**监控时间窗口**
- 竞价阶段：9:15 - 9:25（有效时段内触发）

**配置参数**
- 涨跌幅阈值：默认 5%（高开/低开幅度）

**播报格式**
> `竞价预警：某某股份高开4.2%`
> `竞价预警：某某股份低开6.1%`

---

### 3.8 止盈/止损提醒

**配置方式**
- 用户在自选股上设置止盈价（高于成本价）和止损价（低于成本价）
- 触发后通过通知告知用户，不自动操作

**播报格式**
> `⚠️ 茅台触及您设定的止盈价1700元，当前1698元`
> `⚠️ 茅台触及您设定的止损价1650元，当前1645元`

---

### 3.9 后台运行能力

**iOS 实现**
- `UIBackgroundModes: audio` — 持续后台音频播放
- `AVAudioSession` 配置为 playback 模式
- Foreground Service 保活
- 锁屏时通过本地通知 + TTS 语音播报

**Android 实现**
- Foreground Service + 持久通知栏
- `android.permission.FOREGROUND_SERVICE`
- `android.permission.WAKE_LOCK` 防止休眠

**用户体验**
- 退到后台 → 状态栏显示"听股通运行中"
- 锁屏 → 继续播报
- 与音乐播放器后台逻辑完全一致

## 4. 数据架构

### 4.1 数据源抽象层

```
┌─────────────────────────────────┐
│      DataSource Interface        │  ← 统一抽象接口
├─────────────────────────────────┤
│  - getStockQuote(code)            │
│  - streamAllQuotes()               │
│  - getSectorList()                 │
│  - streamSectorQuotes()            │
│  - getIndexQuote(index)            │
│  - getPreQuote(code)               │  ← 集合竞价数据
│  - getLimitUpPrice(code)          │  ← 涨停价计算
└────────┬──────────────────────────┘
         │
    ┌────┴────┬──────────┐
    ▼         ▼          ▼
┌────────┐ ┌────────┐ ┌────────┐
│东方财富 │ │ 万德   │ │ 预留   │
│Provider│ │Provider│ │Provider│
└────────┘ └────────┘ └────────┘
    Phase 1   Phase 2   扩展
```

**接口设计原则**
- 所有数据源实现统一 Interface
- 业务层不感知具体数据源
- 更换数据源只需替换 Provider，不改业务逻辑

### 4.2 实时数据流

```
东方财富 WebSocket 全市场实时行情
    │
    ▼
DataSource Adapter（统一格式转换）
    │
    ▼
行情数据模型（StockQuote / SectorQuote / IndexQuote）
    │
    ├─────────┬──────────┬──────────┬──────────┐
    ▼         ▼          ▼          ▼          ▼
自选股    拉升/下跌   涨停/跌停/   板块异动   大盘异动
判断层    判断层      炸板判断层   判断层     判断层
    │         │          │          │          │
    └─────────┴──────────┴──────────┴──────────┘
                        │
                        ▼
              播报调度器（Scheduler）
              ┌──────────────────┐
              │ · 优先级排序      │
              │ · 去重判断        │
              │ · TTS 队列管理    │
              │ · 打断/恢复逻辑   │
              └────────┬─────────┘
                       │
                       ▼
                 TTS 语音播报
                 本地通知（锁屏时）
```

## 5. 播报调度器设计

### 5.1 核心状态机

```
状态：IDLE（无播报）| PLAYING（播放中）| QUEUED（有队列）

触发事件：
  - 行情数据到达
  - 用户开启某播报类型
  - TTS 播放完成

响应规则：
  IDLE  + 触发 → 立即播放，状态=PLAYING
  PLAYING + P0/P1触发 → 打断当前，插队播放（P0优先于P1）
  PLAYING + P2/P3/P4触发 → 进入队列，状态=PLAYING+QUEUED
  PLAYING + 播放完毕 → 出队下一条，状态=IDLE或PLAYING
```

### 5.2 被打断恢复机制

- P0/P1 打断当前播报时，记录被打断内容的"断点"
- 打断内容播放完毕后，从断点继续（不从头开始）
- 若打断的是自选股轮播，则记录被打断的股票编号，恢复后从下一只继续

## 6. 页面清单

### 页面 1：首页（Dashboard）
- 自选股列表（名称 / 价格 / 涨跌幅 / 播报状态）
- 后台运行状态指示（绿色呼吸灯）
- 实时预警气泡（涨停/炸板/拉升等紧急播报出现时）
- 板块异动摘要（今日已触发列表）
- 快速添加股票入口

### 页面 2：自选股管理
- 搜索并添加股票（名称 / 代码 / 拼音）
- 列表展示所有自选股
- 单只股票设置：播报间隔、止盈价、止损价
- 左滑删除

### 页面 3：监控设置（核心页面）
- Tab 1「播报开关」：11 种播报类型卡片，每张独立开关
- Tab 2「参数配置」：已开启播报类型的阈值配置
  - 拉升阈值（滑块，默认3%）
  - 下跌阈值（对称拉升阈值）
  - 大盘异动阈值（默认1%）
  - 成交量倍数（默认3倍）
  - 集合竞价阈值（默认5%）
- 数据来源说明（东方财富 Phase 1 / 万德 Phase 2）

### 页面 4：个人中心
- 用户信息
- 通知管理 / 数据缓存
- 隐私政策 / 用户协议
- 版本信息

## 7. 技术规格

| 维度 | 技术选型 | 说明 |
|------|---------|------|
| App 框架 | Flutter | 跨平台 iOS/Android |
